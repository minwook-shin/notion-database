# Security Policy

## Supported Versions

| Version | Supported          | Notes               |
|---------|--------------------|---------------------|
| 2.0.x   | :white_check_mark: | Active maintenance  |
| 1.4.x   | :warning:          | Security fixes only |
| < 1.4.0 | :x:                | No longer supported |

- **2.0.x** — Receives new features, bug fixes, and security patches.
- **1.4.x** — Receives security patches only. No new features or general bug fixes will be backported.
- **< 1.4.0** — End of life. Please upgrade to 2.0.

## Reporting a Vulnerability

Please do **not** open a public issue for security vulnerabilities. Instead:

1. Submit a report privately via **[GitHub Security Advisories](https://github.com/minwook-shin/notion-database/security/advisories/new)**.
2. Include a description of the vulnerability, steps to reproduce, the potential impact, and a suggested fix if possible.
3. You will receive an initial response within **72 hours**.
4. Once the vulnerability is confirmed, a patched release will be published and the reporter will be credited (if desired).
